## P0 follow-up: moe_prefill_down_packed_q{4,6}k_v2 — M_TILE=8 SLM amortization — SHIPPED

Audit (IGC ASM dump, Xe2 simd16 / 128 GRF) showed v1 stage-2 kernels had:
  - 21 load.ugm (DRAM weight reads), 0 load.slm, 0 barriers per launch
  - n_tok outer loop re-streamed each Q-block from DRAM ~22× per launch
  - vs stage 1 (gate_up_silu): 49 load.ugm + 128 load.slm + M_TILE=8 amortization

v2 mirrors moe_prefill_gate_up_silu_q4k's exact SLM pattern:
  - SG_SIZE=16, N_PER_WG=32, WG_ITEMS=512, M_TILE=8
  - SLM A_slm[M_TILE × E_ffn] halfs (8 KiB/WG → 3+ WGs/subslice headroom)
  - Outer loop tiles n_tok in M_TILE chunks
  - Cooperative SLM gather of M packed-rows from h_packed (contiguous; no token-id
    indirection needed, packed rows are already laid out by expert)
  - Inner loop walks blocks_per_col ONCE per tile, dequant per block, multiply
    against M rows from SLM
  - Q4_K dequant inlined verbatim from gate_up_silu (sub<4 vs sub>=4 scale-mid path)
  - Q6_K dequant inlined verbatim from v1 lane lattice
  - SG-reduce per row, lane 0 writes out_packed[(row_base+mm) * H + n]

Compile-time gate: IE_ENABLE_MOE_DOWN_TILE (default ON).
v1 kernels kept callable as fallback when -DIE_ENABLE_MOE_DOWN_TILE=0.

== Audit-vs-actual ==

Audit projected:
  - pk6k 21.3 ms/call → ~5.5 ms/call
  - stage 2 total ~1700 ms → ~460 ms at pp692
  - prefill +30% best-case, +15-25% conservative

Actual delivered: +13.9% to +19.2% across v2-exercising prompts.
Slightly under the audit projection's midpoint, consistent with the
"conservative band" estimate.  The diminishing return at small T
(long-instruction T=80 = +13.9%) vs large T (long-context T=219 =
+19.2%) confirms the win scales with avg n_tok-per-expert (more
amortization opportunity at higher T).

== Architectural takeaway ==

Stage 2 was the largest single prefill lever (~32% of pp512) and is now
~closer to stage 1's per-call cost.  Next prefill levers:
  1. moe_routed stage 1 (gate_up_silu_q4k) — already SLM-amortized but
     still scalar; XMX rewrite was tried in PLAN.md item 11 with 2-SG
     layout (-11-13%); needs wider 4-SG-per-WG design.
  2. moe_shared (Q4_K gate/up + Q6_K down for shared expert) — could
     potentially gain from same M_TILE pattern but shared expert runs
     once per layer (40 calls/forward) so smaller absolute target.
  3. attn_full_block (10 layers × QKV/Out + FA-2 prefill) — small (~8%
     of prefill); needs profiling to see if gemv_q4k_huge XMX dispatch
     hits or falls back.
