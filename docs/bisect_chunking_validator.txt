Tool: `tools/ie_validate_chunking.cpp` → built as `build/tools/ie-validate-chunking`.
Methodology: load model, tokenize /tmp/ppl_test.txt → 1024 tokens.
  Path A: forward(T=1024, start_pos=0) once.
  Path B: forward(T=256) × 4 at start_pos = 0, 256, 512, 768.
  Both paths use fp16 KV cache, fresh state at start.
  Compare per-layer dumped activations at the LAST POSITION (1023 in
  both paths — single's row 1023 vs chunked's chunk-3 row 255).

== Findings ==

1. **L00 (post-embed) and L01-L03 (DN layers) are BIT-EXACT.**
   Embedding lookup, DeltaNet projections, conv1d_silu, alpha+beta gate,
   QKV split + l2_norm, deltanet_recurrence, gated_rms_norm, ssm_out —
   ALL produce identical activations at chunk T=256 vs T=1024.
   (rel_L2 = 0 across all three DN layers → no chunk-size-dependent
   reordering in DeltaNet kernels.)

2. **First non-zero divergence is at L04** = residual after layer index 3 =
   first **FULL-ATTENTION** layer.  max_abs = 3.9e-4 (sub-fp16-ULP for
   activations of magnitude ~1.5).  rel_L2 = 0.24% — small but
   non-zero.

3. **The divergence COMPOUNDS through layers** at ~1.08-1.09× per layer.
   For pure fp16 reordering noise this should be ~1.01-1.03× per layer
   — 9%/layer growth indicates real chunk-size-dependent math, not
   just ULP noise.

4. **Final hidden state (L41, input to lm_head) diverges by 5.5% rel_L2**
   (max_abs = 0.48 on values of magnitude ~113).  After the lm_head
   matmul + softmax, this 5.5% hidden-state divergence yields the PPL
   spikes I measured earlier (PPL 65 on first 50 tokens after T=1024
   prefill vs PPL 4.94 after T=256 prefill).

5. **Bug localized to one of the kernels in the FULL-ATTENTION block:**
   - `gemv_q_T` (Q proj, K proj, V proj, O proj) — uses Q4_K weights
     but DN layers also use these and are bit-exact, so likely NOT
     these.
   - `split_q_gate_per_head` — element rearrangement, DN doesn't use
     this; SUSPECT.
   - `rms_norm_f32w` (QK-Norm) — DN uses different rms_norm variants.
   - `rope_partial` — DN doesn't use RoPE; SUSPECT.
   - `full_attention` — naive WG-per-(t,h) SDPA; SUSPECT.
   - `sigmoid_gate` — element-wise on Q-gate fold-in; DN doesn't use.

== Decision ==

**Bug REPRODUCED** — a single forward(T=1024) call produces a final
hidden-state row 1023 that differs from the equivalent chunked path's
final hidden-state row 255 by 5.5% rel_L2 (0.48 max_abs on magnitude
~113).  This explains the PPL spikes observed in `ie-perplexity` with
prefill_chunk ≥ 512.

The divergence is per-layer ~9% compounding.  Even though L04's initial
divergence is small (4e-4, sub-fp16-ULP), 36 more layers of compound
growth pushes the final state to a regime where lm_head produces
materially different logits.

**Confirmed: production users sending prompts > 256 tokens in a single
batch would receive degraded output for the first ~50-200 generated
tokens until KV cache "settles" past the chunk boundary.**

Production paths (`ie-bench-suite`) chunk at T=256 and don't see this
bug.  `ie-perplexity` with default `--prefill-chunk 256` also doesn't
trigger it.  The bug is hidden behind the chunked-prefill convention.
