## Step 8: memset ws_attn_out_ = 0 at start of attn_full_block — REVERTED

Hypothesis tested: both attention kernels (naive `full_attention` and
tiled `full_attention_fa2_prefill`) might have a write-coverage bug —
they fail to write some output positions, so those positions retain
uninitialized data that varies run-to-run.

Implementation: IE_FA_ZERO_ATTN_OUT compile flag (default OFF).  When
ON: `q.memset(ws_attn_out_, 0, T * N_q * sizeof(half))` before any
kernel writes ws_attn_out_ in the full-attn block.

== Critical observation ==

I have verified bit-exactness of I1-I6 dumps (Q proj, split, QK-norm,
RoPE) ONLY at row 1023 (the last position).  My L03 cross-position
spot check (positions 0..1023) was for A vs B, NOT for A1 vs A2.

The attention kernel READS ALL ctx_len positions from K/V cache when
computing the score.  If positions 0..1022 in the K/V cache have
non-deterministic content (between A1 and A2) but position 1023 is
deterministic, the attention output at row 1023 would diverge — even
though my dumps show I1-I6 EXACT at row 1023.

I have NOT verified that K/V cache contents are bit-exact at ALL
positions between A1 and A2.  This is the MISSING DATA.

== Disposition ==

REVERT — flag default OFF, no production change.
